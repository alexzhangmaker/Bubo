# appNoteCards.html 数据结构文档

本文档整理了知识卡片应用中涉及的核心数据模型，主要包括 IndexedDB 存储结构、任务模型以及业务数据模型。

---

## 1. IndexedDB 存储 (Database: `NoteCardDB`)

系统中包含三个主要的对象仓库 (Object Stores)：

### 1.1 `noteCards` (知识卡片)
存储卡片的基础内容。
- `id`: `number` (自增主键)
- `title`: `string` (卡片标题/问题)
- `content`: `string` (卡片正文/答案，支持 Markdown)
- `tags`: `string[]` (标签数组)
- `quiz`: `Array<{question: string, options: string[], answer: string}>` (内置练习题数组)
- `createdAt`: `number` (创建时间戳)
- `updatedAt`: `number` (更新时间戳)

### 1.2 `tasks` (异步任务)
存储所有通过异步引擎处理的后台任务。
- `id`: `number` (自增主键)
- `requestId`: `string` (唯一请求 ID)
- `businessType`: `'AI_CHALLENGE' | 'AI_SCORING'` (业务类型)
- `input`: `any` (任务输入参数)
- `status`: `'pending' | 'processing' | 'done' | 'failed' | 'cancelled'` (任务状态)
- `result`: `any` (任务执行结果)
- `error`: `string` (错误详情)
- `createdAt`: `number` (创建时间)
- `updatedAt`: `number` (最后更新时间)
- `retryCount`: `number` (重试次数)
- `pollingConfig`:
    - `interval`: `number` (当前轮询间隔)
    - `maxInterval`: `number` (最大轮询间隔)
    - `timeout`: `number` (超时时长)
    - `attempts`: `number` (已尝试轮询次数)

### 1.3 `reviewLogs` (复习记录)
存储用户对卡片的复习历史。
- `id`: `number` (自增主键)
- `cardId`: `number` (关联卡片 ID)
- `reviewTime`: `number` (复习时间戳)
- `duration`: `number` (消耗时长，毫秒)
- `result`:
    - `score`: `number` (掌握程度评分)
    - `stage`: `string` (达到的复习阶段)

---

## 2. 事件系统 (`TaskEvents`)

技术组件通过事件与 UI 进行解耦：
- `TASK_CREATED`: 任务已创建
- `TASK_UPDATED`: 任务状态变更
- `TASK_COMPLETED`: 任务圆满完成
- `TASK_FAILED`: 任务执行失败
- `TASK_CANCELLED`: 任务被取消
- `BULK_UPDATE`: 批量刷新任务列表

---

## 3. 核心业务流程数据流

1. **创建卡片**: `NoteCardManager` -> `StorageService` (`noteCards`)
2. **启动复习**: `ReviewManager` 读取 `noteCards`
3. **AI 对话挑战**: `ReviewManager` 提交任务 -> `TaskManager` -> `StorageService` (`tasks`) -> 轮询服务器 -> 更新 `tasks` -> UI 渲染回复
4. **Quiz 评分**: 用户提交答案 -> `TaskManager` 发起 `AI_SCORING` 任务 -> 获取结果 -> `ReviewManager` 汇总并存入 `reviewLogs`
